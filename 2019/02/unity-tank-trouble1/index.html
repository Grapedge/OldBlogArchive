简介

坦克动荡（Tank Trouble）是一款出色的小游戏，如果你还没有玩过这款游戏，你可以在TANK TROUBLE来体验这款游戏。正好微软学生俱乐部的寒假任务中有制作坦克大战的游戏，我便想仿制一款3D 的 TANK TROUBLE（做3D其实是因为太长时间没碰过2D的内容了，印象中上次碰2D还是刚出Sprite的时候）。下面是本篇博客实现的效果（1.5MB）：

资源的导入

本项目中的资源来自资源商店中的 Tanks！Tutorial，这是一个免费的资源。导入后保留：坦克模型、材质、音效、炮弹和坦克爆炸的粒子效果，然后移除无用的资源。

场景搭建

创建墙壁预设

创建一个 Cube，将其 position 设置为 (0, 1.5, 0)，scale 设置为 (1, 3, 1)，重命名为 Wall，保存为 prefab。创建预设是因为之后会做随机生成地图功能。

通过复制、调整墙壁的 Transform ，完成场景的搭建。

创建坦克预设

从 Models 中找到 Tank 模型，拖入 Hierarchy，将其标签改为 Player 并保存为预制体。

坦克的操控

经过分析，坦克有以下行为：移动、旋转、开火、死亡。
为坦克添加刚体组件，然后创建合适的碰撞器，我推荐碰撞器的大小比合适要略小一些，因为虽然可能会出现轻微的穿模现象，但可以提升玩家的游戏体验。

其他功能

如果你在做坦克对战游戏并找到了这篇博客，你可能想要实现以下功能（这篇博客不会实现，但会给你提供思路）

炮塔旋转

导入模型后可以发现模型分为了若干层次，一般的，你可以在子物体中找到对应的炮塔，你只需要对其进行旋转操作即可。如果你发现并没有绕着想要的点旋转，那么你可以检测编辑器是否切换到了 center，如果是 pivot，那就说明这个建模不是很规范，你可以通过给炮塔创建一个空的父对象，调整父对象到正确的位置，然后旋转父对象带动炮塔即可。

履带移动

你可能会发现上面的实现其实是拖着坦克走，而履带并没有移动。很遗憾，这个坦克模型的履带已经成型，但对于其他的正常的模型来说，你可以找到履带这个物体，然后修改其材质的offset，你可能就会看到你想要的效果。
